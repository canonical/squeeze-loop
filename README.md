# Loops in the Age of AI — the Squeeze Loop Strategy


This repository is the associated repo of technical report [The Squeeze Loop Strategy: Catching Coherent-and-Wrong Artifacts with an Author-Independent Executable Oracle](report.pdf). You can look at [all the editorial reviews](https://github.com/canonical/squeeze-loop/blob/main/claims/editorial_review.md) that lead to the report as it is now.

The repo also holds, the four executable
instances (`src/A`, `src/B`, `src/C`, `src/D`), the evaluation harnesses (`eval/`,
`verify/`), the claim ledger (`claims/ledger.tsv`), and the reading records
(`bib/records/`). The paper's discipline is that **every reported number is
generated by a harness, never typed by hand** — so the paper is recomputable from
this archive.

## The Core Mechanism: The "Squeeze"

Instead of trying to instruct or prompt agents to be better, we propose a structural, organizational solution. The **Squeeze Loop Strategy** traps every agent between two immovable walls:

- **Upper Bound ($U$):** A soft, normative standard (like a human-written specification or handbook) that sets the maximum claim an agent can make.

- **Lower Bound ($L$):** A hard, interpretation-free executable ground truth (like a compiler, test suite, or proof kernel) that the agent cannot alter or game.



### Core Architectural Pillars

- **The Disjointness Principle:** Every agent in the multi-agent loop answers to a completely different pair of upper/lower bounds. Correctness is found purely at the intersection of these bounds, ensuring that one agent's blind spot is caught by another's evidence base.


- **Gate-Defined Done:** An item is only complete when independent, machine-checked gates pass—never when an agent self-reports success.

## Recomputing the numbers in the paper

No number is written into `tex/paper.tex`. Every figure is a LaTeX macro
(`\Res...`) defined in `tex/macros/*.tex`, and each macro file is **generated** by a
script. Recomputing the paper means regenerating those macro files from their
source artifacts and confirming they are byte-identical to what is committed.

### One command (the recompute harness)

The paper ships its own recompute harness — the reflexive instance `src/paper`:

```bash
python3 src/paper/in-band-deliverable/runner/execute_squeeze.py
```

This regenerates all macro files, asserts each is **byte-identical** to the
committed version (`git diff tex/macros` stays empty), rebuilds the PDF, and
reconciles the claim ledger against the manuscript. A green run means every number
in the paper recomputed.

### By hand

```bash
# 1. regenerate every macro from its artifact (Python standard library only)
python3 src/paper/ground-truth/build_ground_truth.py
# 2. confirm reproducibility + build (determinism, pdflatex, citations resolve)
python3 src/paper/ground-truth/verify_ground_truth.py
# 3. build the PDF
cd tex && pdflatex paper && bibtex paper && pdflatex paper && pdflatex paper
```

### Where each number comes from

| Macro file(s) | Generated by | Reads |
|---|---|---|
| `results.tex`, `results_b.tex`, `results_c.tex` | `src/{A,B,C}/evidence/measure_squeeze.py` | each warehouse/app + seeded defects |
| `coupling.tex`, `synthesis.tex`, `eval.tex` | `verify/{coupling_measure,synthesis,eval_protocol}.py` | the A/B/C results |
| `pilot.tex`, `pilot2.tex`, `pilot3.tex`, `study.tex`, `swebench.tex` | `eval/{pilot,pilot2,pilot3,study}/score.py`, `eval/swebench/run_protocol.py` | committed run artifacts |
| `levelup.tex` | `verify/levelup_measures.py` | the four `src/*/diversity/diversity_results.json` |
| `reflexive.tex` | `verify/reflexive_measures.py` | the ledger, reading records, and per-circle spec docs (self-counts) |

### Dependencies

- **Python 3 (standard library only)** and **`pdflatex`/`bibtex`** are enough to
  recompute **every number currently in the paper** and rebuild the PDF. The
  warehouses/apps under `src/*/ground-truth/shared/` are git-ignored and rebuilt
  deterministically by each instance's `build_ground_truth.py`; the evaluation and
  diversity JSONs are committed, so the macros recompute from them with no extra
  tooling.

Two things need more, and only to re-run the underlying **trials** (not to
recompute the macros from the committed artifacts):

- **Coq/Rocq** — to re-run Use Case D's kernel squeeze and its diversity trial.
  `src/D/run_in_lxc.sh` provisions it in a container, or `opam install coq`. The
  diversity trials are **seeded**, so they reproduce.
- **SWE-bench's per-repository Docker harness** — for the real-bug step, which the
  paper marks as wired-but-pending.

### The guarantee

`verify_ground_truth.py` regenerates the macros and **fails if any committed number
differs by a byte**. So "recompute the numbers" and "verify the numbers" are the
same command — which is the reflexive point of `src/paper`. See
`src/paper/README.md` for the full reflexive-instance description and
`docs/containers.md` for how the use-case agents run in unprivileged LXC
containers.

## Running the paper on itself

Beyond recomputing the numbers, the paper is *developed* by an AI coding agent running
the squeeze loop on the manuscript — the "paper-impl circles" of `paper-impl.md`. To run
the paper on itself, a good prompt is:

> **run a paper cycle and share the judgment on the paper itself.**

The agent runs the reflexive squeeze (the recompute harness plus the connective-tissue,
self-containment, Gate S, and reflexive-monitor checks) and then gives an editorial (Gate
A) judgment of the manuscript's current state — strengths, weaknesses, and what to do
next — rather than rubber-stamping a green run.

The same cycle can be *steered* by naming a specific review goal — the agent then runs the
machine gates plus a **disjoint** review (a different model than the author, so it does not
share the author's blind spots) focused on that goal, applies the fixes, and re-verifies.
Another good prompt, which audits the manuscript's narrative against an explicitly stated
"global story":

> **Read the full paper once more for consistency using the paper cycle. Make sure that each
> paragraph is coherent and why it is here, like a small part of a global story. Do not
> hesitate to change the introduction or/and conclusion of a section to reflect that, if you
> think a link with the global story is missing. The global story is: this paper is a generic
> framework which explains how to use AI to create a link between a soft truth and the concrete
> (hard truth) world. This global story is the implicit upper bound of the paper cycle.**

This treats the stated global story as the cycle's *upper bound* (the soft truth the
manuscript must answer to) and the manuscript itself as the lower bound — a squeeze applied to
the paper's own narrative. The agent reads every section against that story, flags paragraphs
that do not earn their place (orphans, buried ledes, non-sequiturs) and sections whose opener
or closer never connects them to the whole, and rewrites those openers/closers to thread the
link — adding it only where it is genuinely missing and a distinct point, never as a repeated
refrain. Other goals can be swapped into the same form: *"make sure each term is well defined
and introduced before use"*, *"run a summary-vs-body faithfulness sweep"*, or *"make this
qualifier consistent everywhere."* These recurring review goals are catalogued as standing
sweeps in the `write-a-paper` skill (`config/skills/write-a-paper/`), which the agent proposes
proactively while iterating.
