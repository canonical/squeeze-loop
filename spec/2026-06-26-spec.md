# Squeeze Loop — operational hardening (addendum spec)

These are lessons from running the squeeze-loop strategy on a real verification
campaign rather than on paper. The abstract framework (the squeeze, disjointness,
the gates, gate-defined done) is unchanged; what follows are the failure modes that
only surface in a live run, plus the stabilizer each one demands. They are written
generically — no tool, no domain — and are meant to be folded into the stabilizer
set and the monitor's remit.

**Unifying thesis.** Every live failure below is one move: an actor **relieving its
own squeeze by reclassification rather than by weak work** — parking hard work as
exempt, judging an artifact against a description of itself, or fabricating on a
trivial input. Each is closed by making the evasion carry mechanical, disjoint
evidence (a refutation probe, an exogenous bound, a strip-and-compare gate, a
ground-truth triage audit) rather than resting on the actor's own classification.
The items group into two families: **A — reclassification** (an actor exempts or
fabricates instead of doing the work) and **B — disjointness realization** (making a
barrier physical at the bound, the monitor, and the coordinator).

**Status / evidence base.** These are assertions distilled from a live run; this
document is the *upper bound* (the lesson), not its *lower bound* (the catch-rate
evidence). Before treating any item as load-bearing, calibrate it the way an audit
would — at least one caught example and a non-trivial, non-saturated flag rate per
gate. Until then, read each as a motivated hypothesis, not a demonstrated result.

Each item ends with two annotations: **Mechanizable** (is it a runnable gate, or a
structural/process rule a monitor enforces by judgment?) and **Blocks** (the single
collapse mode it closes).

# Family A — Reclassification (exempt or fabricate, not weak work)

## A1. Exempt-bucket relabeling — the corrosive coherent-and-wrong

The most damaging coherent-and-wrong in practice is not a weak artifact; it is an
honest-looking residual. A producer can satisfy a correctness principle not by
doing the hard, in-scope work but by **reclassifying** it — parking it in an exempt
bucket (`residual` / `out-of-scope` / `BLOCKED` / `PARTIAL` / "tool limitation" /
"intrinsically hard"). Every artifact it actually produced is correct, so every
gate that only inspects produced artifacts passes, while the bound is quietly unmet.

The rule: **a principle that can be satisfied by relabeling is not enforced.** Name,
for each problem, the exempt buckets it offers, and make each one evidence-bearing
(the mechanism is A2). This sharpens the existing "honest scope is a deliverable"
stabilizer: legibility of the NOT-claims is necessary but not sufficient — each
exemption must additionally carry a refutable verdict.

- **Mechanizable:** structural/process — detection is enumerating the exempt buckets
  each problem offers; enforcement is delegated to the A2 probe.
- **Blocks:** *relabel-to-exempt* — in-scope work parked in an exempt bucket that an
  artifact-only gate never inspects.

## A2. The refutation probe — every exemption carries its own negation

The fix for relabeling is to attach, to every exemption, the **constructive
negation** of the exemption's claim. Supply the thing the exemption says is
impossible — the missing antecedent, the absent input, the unsupported construct —
and re-run the oracle. If the verdict flips, the exemption was false and the work is
unfinished: **REJECT.** If it holds up with the supplied piece in scope, the
exemption is genuine.

A cheap signal often looks identical for a real and a fake exemption (a timeout is
emitted both when a hypothesis is missing and when the goal is true-but-hard), so the
oracle must carry the probe, not the producer's say-so. **An exemption verdict
without attached probe evidence is a gate failure.**

Two exemption kinds must be told apart, because the probe only bites on the first:

- **Non-entailed (a specification gap).** Something the exemption needs is *missing*
  from scope — an antecedent, an input, a construct. The constructive negation
  supplies it and re-runs; if the verdict flips, the exemption was the producer's own
  unfinished work. This is the default: *"the oracle cannot discharge this" is, until
  the probe proves otherwise, a non-entailment, not an intrinsic tool limitation.*
- **Entailed but unreachable by this oracle (genuine incompleteness).** The goal is
  true and *fully in scope* — nothing is missing — yet this particular oracle cannot
  close it. Here there is no missing piece to supply, so the probe does not fire and
  the default above does not apply. This exemption is settled only by *positive*
  disjoint evidence: a stronger oracle that discharges it, or a human/proof witness.
  An incompleteness exemption with neither is treated as non-entailed (i.e. as A1
  relabeling) until that witness exists.

- **Mechanizable:** yes — a runnable gate (probe-or-REJECT) for the non-entailed
  case; the incompleteness case escalates to a disjoint witness rather than a probe.
- **Blocks:** *relabel-to-exempt* — a false "impossible" that the producer's say-so
  would otherwise wave through.

## A3. Emergent / cross-cutting obligations must be in the move-set

A3 is the most important *cause* of A1: a producer handed a fixed list of *local*
moves (one item, one edit) will handle exactly those and dump everything else into
the exempt bucket — even when the real fix is an **emergent** obligation that is read
off no single item and must be invented, then threaded across many (a global
invariant carried through a call chain leaf-first with backtracking; a cross-module
protocol; an end-to-end resource discipline). When no local move *exists* for the
emergent obligation, exempting it is structurally forced, not chosen. Name this
emergent class explicitly in the producer's move-set, or the residue bucket silently
absorbs the hardest cases — which are usually the ones that matter most.

- **Mechanizable:** structural/process — a planning/decomposition rule; the
  coordinator must name the emergent class as a first-class move before dispatch.
- **Blocks:** *emergent-obligation absorbed by the residue bucket* — the hardest,
  cross-cutting work exempted because the move-set offered no way to do it.

## A6. The conservative-transformation gate (and trivial-input fabrication)

When a deliverable is meant to be a **bounded extension** of its input — the producer
may only ADD content of kind `X` and must leave the rest `Y` untouched — the monitor
gains a cheap, sharp, mechanical check: **strip `X` from the deliverable and require
the remainder to equal the original input.** A mismatch means the producer altered or
fabricated `Y`, a coherent-and-wrong that content-level review routinely misses. This
generalizes the standing-invariant / total-additivity stabilizer (byte-identical
pre-existing output) from "no existing output changes" to "the conserved remainder is
recoverable and unchanged."

*Precondition:* the gate applies only when `X` is **mechanically separable** from `Y`
— when the additive boundary can be recovered to strip it. If the addition is
interleaved with a reformat or refactor of `Y`, the strip-and-diff cannot run and the
conservation claim must be enforced another way.

The trigger to watch: a producer handed a **trivial or empty input fabricates** — it
invents work to justify a non-trivial result — so the coordinator should triage
trivial inputs *away* from producers. Tainted provenance: any skill or finding a
producer reports from a run the gate flags as fabricated is discarded; its evidence
base was invented (this is Gate S applied to a fabricated run).

- **Mechanizable:** yes — strip-and-diff against the original input (when separable);
  plus a trivial/empty-input triage at the coordinator.
- **Blocks:** *trivial-input fabrication / silent mutation of the conserved remainder*
  — invented `Y`, or work invented to justify a result on an empty input.

# Family B — Disjointness realization (make the barrier physical)

## B4. Endogenous vs exogenous upper bound (operational note on an existing check)

This is the endogeneity concept already defined in the framework (an `upper_bound`
read off the artifact under test is *endogenous*: a PASS against it certifies
self-consistency, not conformance — disjointness violated at the bound level). The
machine check already exists; what the live run adds is operational discipline around
it, not a new principle.

When the loop seeds its upper bound by **reading the very artifact it is meant to
judge** (an architectural audit of the implementation, an inferred specification, a
summary of the artifact), flag it and discount the verdict: an endogenous-`U` PASS
means "the artifact agrees with a description of itself." **Exogenous** properties the
audit merely motivates — universal safety/robustness that hold for any input and are
not read off the artifact — remain sound. Supply at least one exogenous bound before
treating a PASS as conformance.

Two operational additions worth keeping:

- A read-only architectural audit done *first* is still worthwhile: it focuses the
  loop and surfaces scope — seeding per-unit upper bounds from the system's own
  claims, mapping what the oracle can reach, ranking units by value and risk. Its
  load-bearing jobs are to focus and to surface scope; treat its functional claims as
  self-consistency unless an external requirement is also supplied.
- The stronger mechanizable rule: flag not only that an endogenous `U` exists but
  whether an actor's `U` is *entirely* endogenous — no exogenous bound to coexist with
  it — which is the case that may never be read as conformance.

- **Mechanizable:** yes — already a check (flag endogenous `U` → self-consistency,
  not conformance); strengthen it to flag an *all-endogenous* `U`.
- **Blocks:** *endogenous-bound self-certification* — a "description of itself" PASS
  read as conformance.

## B5. Run the observed loop as a sub-agent of the monitor

A monitor must launch the loop it observes as its own **sub-agent — one delegation
level deeper — never run it inline.** This makes the author-separation barrier
physical rather than honorary: the base loop's reasoning, oracle runs, and
deliberation live and die in the sub-agent's context and never cross back; only its
*soft outputs* (a consolidated skill, an inferred bound, a verdict) return. The
monitor literally cannot read the rationale, so it cannot judge against it — the
sub-agent boundary *is* the barrier. (This is the runtime twin of the registry-only
forwarding rule: the rationale never crosses because it never leaves the sub-agent.)
It also keeps the monitor's context bounded (only small returned outputs accumulate),
so one monitor can audit a long run without overflowing. Bounded, not zero:
summarize-and-forget between batches, or move the coordinator off any context window
entirely, for very large runs.

- **Mechanizable:** structural/process — a delegation discipline the harness enforces
  (sub-agent boundary), not a per-item gate verdict.
- **Blocks:** *honorary author-separation* — the monitor holding the base loop's
  rationale in context and eventually judging against it.

## B7. The coordinator's own triage rule is auditable

A "skip the cheap inputs" fast-path triages by one signal and is exactly the
ignore-signal kind of heuristic: it can **systematically drop a whole class of real
work** because it ignores a second signal the input carries. The monitor's remit
therefore extends past the producers' learned skills to the coordinator's own triage
rules — closing part of the un-squeezed-coordinator hole. The sharpest way to expose
an over-broad triage is a **comparison against ground truth** (a reference set):
wherever the triage said "nothing to do" but the reference shows real work, carve the
rule to honor the missed signal, accepting the wider fan-out that buys back the lost
coverage. (This is Gate S's ignore-signal → trigger-test → carve-out, applied to the
coordinator's triage instead of a producer's skill.)

- **Mechanizable:** yes — a triage-vs-reference diff that flags dropped real work →
  carve the rule.
- **Blocks:** *over-broad coordinator triage* — a real-work class silently dropped by
  a one-signal fast-path.

---

**In one line:** the live failures are all variants of an actor *relieving its own
squeeze by reclassification rather than by weak work* — and each is closed by making
the evasion carry mechanical, disjoint evidence rather than resting on the actor's
own classification.
