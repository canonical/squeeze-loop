METHOD (three sentences)
The authors define an indirect prompt-injection threat model in which adversarial instructions are embedded in external content (web pages, emails, documents) that an LLM-integrated application later retrieves and processes, alongside direct first-party injection. They develop and demonstrate working proof-of-concept exploits against real and synthetic systems — including Bing Chat (GPT-4-backed), GPT-4 code completion, and LangChain-style agents with tool/API access — covering remote control, persistence, data exfiltration, and unauthorized API invocation. They organize the demonstrated impacts into a six-class threat taxonomy (information gathering, fraud, malware, intrusion, manipulated content, availability) and release attack prompts and demos.
LIMITATIONS
The work is primarily demonstrative (proof-of-concept exploits) rather than a quantified success-rate benchmark, and the strongest novel emphasis is on indirect (retrieved-content) injection, whereas Use Case B's refund bot faces mainly direct conversational adversaries; specific exploits target 2023-era models/systems and depend on their integration details, so applicability to a given hardened deployment must be argued, not assumed.
CONTRIBUTION (why cited in THIS paper; which claim it supports; verdict)
Cited as the canonical real-world study showing that LLM-integrated applications which take consequential actions (API calls) can be driven into attacker-chosen behavior by adversarial prompts that "override original instructions and employed controls" — directly the threat model of an action-taking refund agent that could be manipulated into a money-out action. It grounds the manuscript's argument that authored policy alone is insufficient and that consequential actions need an external enforcement boundary, motivating Use Case B's REST action lockpoint.
